# Incident Response Runbook

## Purpose
Standardize detection, triage, containment, recovery, and reporting for security and availability incidents.

## Roles
- Incident Commander (IC)
- Security Lead
- SRE Lead
- Comms Lead

## Severity Levels
- SEV1: Data breach, large outage, compliance impact
- SEV2: Policy enforcement failure, partial outage
- SEV3: Degraded performance, minor incidents

## Workflow
1. Detect (alerts, audits, customer report)
2. Triage (severity assessment)
3. Contain (limit impact)
4. Recover (restore services)
5. Report (postmortem, compliance evidence)

## Evidence
- Audit logs
- Governance logs
- Incident tickets
- Root cause analysis
